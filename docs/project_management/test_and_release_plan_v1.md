## Топология команды и инфраструктура

**1. Инфраструктура QA / Тестирования (удаленный компьютер)**
- **Специалист:** `loginovfedor`
- **Окружение:** Свой компьютер, своя копия Git-репозитория, свой инстанс Unity и свой локальный агент Codex.
- **Роль:** "Начинающий разработчик" / Тестировщик (Playtester, QA).
- **Зона ответственности:** Поиск missing-ссылок, ручной прогон геймплейных механик, субъективная оценка читабельности UI и управления на разных платформах.

**2. Инфраструктура Разработки (текущий компьютер)**
- **Специалист:** `xlabkm-ux` (Owner)
- **Окружение:** Текущий ПК. Основной агент `Codex`.
- **Роль:** Ведущий программист и архитектор.
- **Зона ответственности:** Фикс багов, написание тестов, стабилизация кода, настройка пайплайнов отказоустойчивости и формирование Release Candidate.

## Распределение задач из GitHub Product Backlog

### Блок 1: Задачи Разработки (Выполняется `xlabkm-ux` на текущем ПК)

#### BWF-036 - Combat and perception regression sweep
- **Исполнитель:** `xlabkm-ux` с привлечением агента `Codex`.
- **Суть:** Прогон автоматических тестов (combat, vision, alert escalation). Агенты фиксируют падения и исправляют C# логику и графы при необходимости.

#### BWF-037 - Hostage success/fail scenario sweep
- **Исполнитель:** `xlabkm-ux` с привлечением агента `Codex`.
- **Суть:** Покрытие юнит-тестами логики заложника и зон эвакуации. Codex пишет недостающие тесты и исправляет код.

#### BWF-038 - Save/load resilience sweep
- **Исполнитель:** `xlabkm-ux` (ведущая роль) + агент `Codex` (помошник).
- **Суть:** Сложнейшая техническая задача по проверке отказоустойчивости сохранений (JSON-схемы, recovery-логика, fallback behavior). Выполняем мы с вами в паре.

#### BWF-039 - Localization coverage sweep
- **Исполнитель:** `Codex` (оркеструет `xlabkm-ux`).
- **Суть:** Автоматизированная проверка покрытия словарей и key-fallback с помощью скриптов поверх JSON-файлов локализации. 

#### BWF-042 - v1 candidate baseline
- **Исполнитель:** `Codex` + `xlabkm-ux`. Одобряет финально `loginovfedor`.
- **Суть:** Сведение итогов по всем Workflow. Блокируется до успешной приемки QA. Подготовка итогового релиз-кандидата.


### Блок 2: Задачи QA и геймплейной приемки (Выполняется `loginovfedor` на втором ПК)

#### BWF-035 - Reference integrity sweep
- **Исполнитель:** `loginovfedor`.
- **Суть:** Визуальная и физическая проверка всех сцен и префабов в редакторе Unity на наличие битых ссылок (Missing Reference), а также проверка логов редактора во время старта миссий.

#### BWF-040 - Android perf/readability quick pass
- **Исполнитель:** `loginovfedor`.
- **Суть:** Сборка под Android (или запуск в симуляторе нужного Quality Profile). Оценка того, не съехал ли UI, читаются ли элементы и нормально ли запускается.

#### BWF-041 - Windows readability/input sanity pass
- **Исполнитель:** `loginovfedor`.
- **Суть:** Десктоп-проверка. Прогон управления "как игрок". Проверка InputCompat. Читабельность подсказок, оверлеев прицеливания и зон шума.
